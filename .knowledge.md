# Franchise.id - Project Knowledge

This project is a high-performance static site converted from a WordPress/Elementor codebase, hosted on **Cloudflare Pages** for free, utilizing **Cloudflare Functions** and **Google Sheets/Supabase** as backend services.

### Core Architecture
- **Static Hosting:** Cloudflare Pages (Free Tier).
- **Serverless Backend:** Cloudflare Functions (located in `/functions/`).
- **CMS / Data Source:** Google Sheets (Spreadsheet ID: `1p3Ke25SYZx0Yanv2MHy73eCbK_jE-qgzVC7ue5Tu3mU`).
- **Secondary Database:** Supabase (for complex relational data and operations).
- **Asset Optimization:** Cloudinary for automatic image transformations (`e_bgremoval`, `c_pad`, `f_auto`, `q_auto`).
- **Static Generation:** Build-time scripts (`js/build-listing.js`, `js/build-details.js`) generate HTML from Google Sheets data.

### Data Sources (Google Sheets)
The project relies on three primary tabs in the Google Spreadsheet:
1.  **FRANCHISOR:** Core directory of franchise brands with full investment and contact details.
2.  **FRANCHISEE:** Database of leads and potential investors who have filled out interest forms.
3.  **UNCLAIMED:** Large dataset of potential franchise listings to be claimed or imported.

### Build & Deployment
- **Site Generation:** Static HTML is generated from templates in `/templates/` using Google Sheets data.
- **Form Handling:** Leads are captured via `functions/form-submit.js` and stored in Google Sheets/Supabase.
- **Caching:** API responses from `get-franchises.js` are cached for 1 hour for performance.
