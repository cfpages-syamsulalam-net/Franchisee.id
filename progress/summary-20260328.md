# Project Progress Summary: Franchise.id
Date: 2026-03-28

## 1. Project Architecture & Foundation
- **Tech Stack:** Successfully transitioned to a **WordPress-to-Static** architecture hosted on **Cloudflare Pages**.
- **Data Flow:** Uses **Google Sheets** as a CMS, with **Cloudflare Functions** handling dynamic form submissions and **GitHub Actions** (Node.js scripts) serving as the SSG engine.
- **Documentation Standards:** Established `GEMINI.md` as the Single Source of Truth, `TECHNICAL_INVENTORY.md` for logic tracking, and `FORM_SCHEMA.md` for data integrity.

## 2. Completed Features & Milestones
- **Hybrid SSG Listing:** The main listing page now merges "Verified" franchisors with "Unclaimed" brands, significantly increasing SEO depth.
- **High-Performance Search:** Implemented a static `unclaimed-brands.json` for instant, zero-latency autocomplete in the registration forms.
- **Real-Time Sync:** Developed a Google Apps Script trigger that allows business owners to sync Sheet changes to the live site via GitHub Repository Dispatches.
- **Advanced Form Logic:** The registration form (`js/form-franchise.js`) includes real-time BEP (Break-Even Point) and ROI calculations, multi-step validation, and direct Cloudinary image uploads.
- **SEO Optimization:** Fully implemented JSON-LD structured data, dynamic breadcrumbs, and an automated sitemap generator for all ~500+ detail pages.
- **Build Optimization:** Enhanced GitHub Actions with hash-based change detection to skip redundant builds and added a 15-minute polling fallback.

## 3. Current Focus: The Claiming Workflow
- **Status:** **Phase 5 (Unified Claiming Workflow)** is currently **In Progress**.
- **Goal:** Seamlessly merge the "Claim Brand" tab into the main "Franchisor" registration flow.
- **Mechanics:** When a user selects an unclaimed brand, the system pre-fills the Franchisor form with known data (brand name, category, investment) and injects a hidden `unclaimed_id`.
- **Backend Integration:** The `form-submit.js` function is being finalized to handle "post-claim cleanup," which migrates the brand from the `UNCLAIMED` list to the active `FRANCHISOR` database upon successful submission.

## 4. Technical Health & Safety
- **Logic Recovery:** Successfully refactored and restored critical business logic (BEP calculations, progress bars) into `js/form-utils.js` to keep the codebase clean.
- **Safety Protocols:** Established strict "surgical edit" rules (using `replace` instead of `write_file`) for large files to protect legacy Elementor structures and complex JS logic.

## 5. Recent Activity (March 2026)
- Consolidated project knowledge into a centralized `KNOWLEDGE.md`.
- Normalized the `CHANGELOG.md` format for better traceability.
