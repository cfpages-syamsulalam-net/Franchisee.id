## Important behavior
- OCR results are still review-only. Successful OCR writes to `franchise_asset_knowledge` and `proposal_extraction` suggestions, not directly to canonical listing fields.
- Running all 100 queued jobs in one browser request is intentionally avoided because provider calls can timeout or burn quota unexpectedly.
- The protected OCR worker drains pending jobs only; it does not enqueue assets by itself.
- Scheduled worker runs are not enabled until `OCR_WORKER_ENABLED=true` is set as a GitHub repository variable. Manual workflow runs remain available for one-off testing.
- Provider rate-limit metadata is a local safety guard. It complements provider-side enforcement and existing daily/monthly quota counters; it does not replace periodic verification of provider terms.
