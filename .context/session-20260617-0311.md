# Session Snapshot - 2026-06-17 03:11 Asia/Jakarta

## Focus
- Implemented explicit Clerk/D1 bidirectional sync for the franchise network account system.
- D1 remains the authorization source of truth; Clerk remains the identity/session source of truth.

## Code Changes
- `functions/_clerk-auth.js`
  - Upserts D1 users from Clerk Backend API user objects and Clerk webhook JSON payloads.
  - Syncs D1 role/status snapshots into Clerk `publicMetadata.franchiseNetwork` and `privateMetadata.franchiseNetwork`.
  - Exposes helpers for verified webhook sync, role assignment/removal, D1 user lookups, metadata backfill, and soft deletion after Clerk user removal.
- `functions/clerk-webhook.js`
  - Verifies Clerk webhooks with `CLERK_WEBHOOK_SIGNING_SECRET`.
  - Handles `user.created`, `user.updated`, and `user.deleted`.
- `functions/auth-config.js`
  - Reads `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` with `CLERK_PUBLISHABLE_KEY` fallback.
- `functions/user-role.js`
  - Admin-only endpoint for assigning/removing `franchisee`, `franchisor`, `staff`, and `admin` roles in D1.
  - Syncs the resulting D1 role snapshot back to Clerk metadata immediately.
- `functions/sync-clerk-metadata.js`
  - Admin-only repair/backfill endpoint for one D1 user or up to 500 D1 users.

## Operational Contract
- Clerk -> D1:
  - Configure Clerk webhook endpoint: `https://franchisee.id/clerk-webhook`.
  - Subscribe to `user.created`, `user.updated`, and `user.deleted`.
  - Set Cloudflare Pages secret `CLERK_WEBHOOK_SIGNING_SECRET`.
  - Set Cloudflare Pages variable `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` and secret `CLERK_SECRET_KEY`.
- D1 -> Clerk:
  - Use `POST /user-role` for normal role changes so D1 and Clerk metadata stay aligned.
  - If D1 is changed manually through SQL, run `POST /sync-clerk-metadata` afterward.
- Clerk metadata is a routing/UI cache only. Server authorization must continue to query D1 roles.
