# Session Snapshot - 2026-06-19 21:21 (Asia/Jakarta)

## Focus
Fix successful Cloudflare Pages deployment where CSS/JS/images were missing because `dist` only contained Astro-generated `/peluang-usaha` pages.

## Diagnosis
The project is a hybrid migration:
- Legacy WordPress-exported static HTML/assets still power most routes and styling.
- Astro currently generates only D1-backed `/peluang-usaha` pages.

Cloudflare Pages now serves `dist`, but Astro's `dist` output did not include legacy `wp-content`, `wp-includes`, root HTML pages, `css`, `js`, images, and other static export directories. Browser 404s for Elementor/Astra/WordPress CSS and JS were expected from that incomplete output.

## Change
- Added `scripts/copy-legacy-static.mjs`.
- Updated `build:astro` to run `astro build` and then `pnpm run copy:legacy-static`.
- The copy step copies legacy public files/directories into `dist` without overwriting existing Astro output.
- The copy step skips top-level `peluang-usaha` so Astro owns the D1-backed directory route and legacy folder indexes do not compete with flat `.html` pages.

## Verification Target
After `pnpm run build`, `dist` should contain:
- Astro-generated `dist/peluang-usaha/index.html` and `dist/peluang-usaha/*.html`.
- Legacy static assets such as `dist/wp-content`, `dist/wp-includes`, `dist/css`, `dist/js`.
- Legacy static routes such as `dist/index.html`, `dist/daftar`, `dist/login`, and category/article directories.
