# Session Snapshot - 2026-07-13 10:34 Asia/Jakarta

## Focus
The user needed Review OCR / proposal suggestions to show trustworthy proof on hover and support granular approval. Example: approve every extracted field except `royalty_basis = none`.

## Implemented
- Granular approval:
  - `js/dashboard-review.js` renders an admin-only checkbox per suggested field, checked by default.
  - Approving sends `approved_fields` to `/dashboard-data`.
  - `functions/_dashboard-schemas.js` validates `approved_fields` against shared editable listing fields.
  - `functions/_dashboard-actions.js` applies only selected fields, records skipped fields in review notes/audit metadata, and queues rebuild metadata with selected/skipped fields.
- Proof hover:
  - OCR bundle evidence remains read from `old_value.__ocr_evidence`.
  - Older `proposal_extraction` or OCR suggestions that lack embedded evidence are enriched at read time by `functions/_dashboard-review-evidence.js`.
  - The helper reads `franchise_asset_knowledge` plus `franchise_assets`, matches suggested fields back to structured proposal/OCR rows, sanitizes source excerpts, and attaches evidence under `old_value.__ocr_evidence`.
  - The Review UI labels image proof as `Bukti dokumen` and uses the existing `data-ocr-image-preview-url` hover preview component.
- Maintainability:
  - Evidence matching was extracted out of `_dashboard-queries.js` into `_dashboard-review-evidence.js` to avoid growing the dashboard read model.
