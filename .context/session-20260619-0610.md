## Focus
Fix Cloudflare Pages deployment failure where Pages skipped the build step, then failed to bundle Functions because npm dependencies were unavailable.

## Trigger
Cloudflare deployment log showed:
- `wrangler.toml` was ignored because it lacked `pages_build_output_dir`.
- No build command was configured, so Cloudflare skipped the build step.
- Pages Functions bundling failed to resolve `@clerk/backend`, `@clerk/backend/webhooks`, and `zod`.

## Changes
- Added `pages_build_output_dir = "dist"` to `wrangler.toml`.
- Added the same Pages output setting and build-command note to `wrangler.example.toml`.
- Added `package.json` script `build`, delegating to `pnpm run build:astro`.
- Added `.node-version` with Node `20.19.4`, matching the Astro 5.x pin.
- Updated AGENTS, AUDIT, CODEBASE, TECHNICAL_INVENTORY, docs/README, D1 static publish strategy, and tech stack decisions to document that Cloudflare Pages must run `pnpm run build` or `pnpm run build:astro` before bundling Functions.

## Cloudflare Pages Setting Still Required
In the Cloudflare Pages project settings, set:
- Build command: `pnpm run build`
- Build output directory: `dist`
- Node version: Node 20.x, if the dashboard exposes an environment variable or runtime setting.

Without the build command, deploy hooks and Git push builds can still skip dependency installation and fail on Pages Functions npm imports.
