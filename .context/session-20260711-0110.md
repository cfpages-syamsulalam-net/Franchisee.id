# Session 2026-07-11 01:10 (Asia/Jakarta)

## Focus
- Implement suggestion 84: prevent normal validation/builds from leaving mass-regenerated D1 bridge HTML source diffs.

## Decision
- Astro now owns deployable public franchise pages in `dist`.
- Normal D1 sync should refresh only:
  - `json/d1-franchise-static-data.json`
  - `json/unclaimed-brands.json`
- Root bridge files under `/peluang-usaha/*.html` and `json/d1-generated-pages-manifest.json` are now legacy/diagnostic output and must be opt-in.

## Implementation
- `scripts/build-d1-franchise-pages.ts`
  - Added `--write-bridge-pages`.
  - Default non-dry run no longer writes root `/peluang-usaha/*.html` or `json/d1-generated-pages-manifest.json`.
  - Default non-dry run still writes the Astro snapshot and claim-search JSON.
  - `--dry-run` still renders bridge pages in memory for validation.
  - Bridge pruning only runs when `--write-bridge-pages` is used.
- `package.json`
  - Added `pnpm run build:d1:franchises:bridge` as the explicit bridge writer.

## Documentation
- Updated `CODEBASE.md`, `TECHNICAL_INVENTORY.md`, `AUDIT.md`, `docs/architecture/TECH_STACK_DECISIONS.md`, `SUGGESTION.md`, and `CHANGELOG.md`.
