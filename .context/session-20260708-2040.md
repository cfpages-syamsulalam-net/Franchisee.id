## Fixes

- Fixed Upstash QStash publish URL formatting.
  - Previous code used `encodeURIComponent(workerUrl)` in the publish path.
  - QStash expects the destination URL as a raw URL path, for example `/v2/publish/https://franchisee.id/ocr-worker`.
  - This likely caused: `invalid destination url: endpoint has invalid scheme`.
- Added scheduler worker URL validation before dispatch.
- Strips a pasted `Bearer ` prefix from the stored QStash token before sending `Authorization: Bearer ...`.
- Added clearer 401 guidance: use `QSTASH_TOKEN`, not signing keys or endpoint URLs.
- Added `retry_ocr_batch_run`.
  - Existing failed/running OCR batches can be rescheduled after fixing scheduler credential or worker URL.
  - Dashboard batch rows now show `Retry` and `Refresh` actions.
- Added automatic scheduler preflight before large OCR batch creation.
  - `start_ocr_batch_run` now blocks before creating `ocr_batch_runs` or assigning jobs if scheduler preflight fails.
  - Upstash QStash preflight publishes a harmless `{ preflight: true }` worker message, so token/URL/worker-secret issues surface before `Jalankan 100` creates a batch.
  - `/ocr-worker` accepts preflight requests without calling OCR providers or consuming OCR quota.
