## Key Decisions Captured
- Free directory remains useful and claimable.
- Premium should justify the yearly price through network distribution, priority placement, rich listing pages, trust verification, lead capture, analytics, media/proposal hosting, and owner convenience.
- Payment should start with manual BCA transfer plus unique code, then move to payment gateway webhook automation.
- Direct BCA API is documented as a later option; avoid bank login scraping or storing bank credentials.
- Premium covers one brand/listing per yearly membership.
- `/premium/` shows the price/payment destination for clarity; `/profil/?tab=membership` generates the exact payable amount with a unique code.
- Admin approval activates one-year premium membership, marks the listing premium, creates missing publication rows for Franchisee.id, Franchise.id, Franchisor.id, and Waralaba.id, publishes those rows, writes audit events, and queues rebuilds.
- Premium readiness currently warns owners/admin reviewers but does not block payment or activation.
- Manual payment details are now dashboard-managed data with a BCA fallback; future QRIS/e-wallet/gateway methods should extend `payment_methods`.
