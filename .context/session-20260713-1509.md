# Session Snapshot - 2026-07-13 15:09 Asia/Jakarta

## Focus
The user identified a data-quality issue in Review OCR: `Total investasi` was suggested as `Rp 119.000.000` from brochure text that actually said `Biaya Kemitraan 119 Juta`. They also asked whether license fee and partnership fee should be separate fields or synonyms.

## Decision
- `fee_license_idr` is the canonical field for the upfront right-to-use/license/partnership/joining fee.
- User-facing label is now `Biaya Kemitraan Awal`.
- Synonyms mapped to that field include franchise fee, license fee, biaya lisensi, biaya franchise, biaya kemitraan, biaya join, joining fee, and investasi kemitraan.
- Ambiguous bundle terms such as `paket kemitraan` are not auto-classified because they may include equipment, stock, renovation, or other components.
- `total_investment_idr` is stricter: it should only come from explicit total/nilai investasi wording, not from a single fee row. Future calculated totals should be a separate reviewed model that can show all included components.
