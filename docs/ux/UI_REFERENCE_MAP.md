# UI Reference Map

Last updated: 2026-07-17 10:22 (Asia/Jakarta)

Use this map before creating or changing public pages, protected app pages, dashboard panels, and generated franchise pages. The goal is visual continuity: new work should borrow an existing Franchisee.id pattern first, then adapt only what the task truly needs.

## Default Visual Sources

| Surface | Primary reference | Use when | Notes |
| --- | --- | --- | --- |
| Public marketing pages | `/premium` | The page needs to persuade, explain value, or drive registration. | Use warm yellow/black/cream branding, direct CTAs, clear value ladder, and no detached generic SaaS shell. |
| Public legal/static information pages | `src/components/LegalPage.astro` plus `/premium` tokens | The page is informational but still public-facing. | Keep Franchisee.id header/footer, readable content width, and high-contrast footer links. |
| Franchise directory pages | `/peluang-usaha` generated by `src/lib/franchise-static.ts` | The user is browsing many opportunities. | Prioritize fast discovery: compact controls, visible listing rows early, clear buyer/franchisor CTAs, and no duplicate search surfaces. |
| Franchise detail pages | Generated `/peluang-usaha/[slug]` output | The user is evaluating one brand. | Show brand facts, contact/action paths, claim CTA, comparison/save actions, and proof-backed data. Do not make it feel like a blog post. |
| Profile/member app | `/profil` shell and `css/profile.css` | Logged-in franchisee/franchisor self-service. | Use warm brand shell, prominent next-best-action panels, and form clarity. |
| Admin/staff dashboard | `/dashboard` shell and compact dashboard cards | Staff operations, review, outreach, publication, Premium, integration setup. | Optimize density and action clarity. Use tabs by job, compact rows, icon controls with shared tooltips, and clear recovery states. |
| OCR/proof review | Dashboard OCR Review and shared tooltip/proof components | Admin must trust structured data suggestions. | Always show source basis, proof preview, granular approval, and conservative no-infer rules. |

## Before Styling A New Surface

1. Pick one primary reference from the table.
2. State the page job in one sentence.
3. Identify the primary user and the next action they should take.
4. Reuse existing colors, spacing rhythm, button shape, card density, and tooltip behavior where possible.
5. If deviating, document why in the relevant tracker before implementation.

## Public Page Copy Guardrails

- Public copy must speak to the user, not to the internal system.
- For buyer pages, the page should help users compare opportunities, save brands, request more info, and register as franchisee members when the next step needs account context.
- For owner/franchisor pages, the page should make the free listing path obvious before Premium. A good base message is: "Punya peluang usaha franchise? Tampilkan brand franchise Anda gratis di Franchisee.id."
- Avoid vague CTAs such as "Pelajari lebih lanjut" when the real action is "Daftar Brand Gratis", "Jadi Member Franchisee", "Bandingkan", or "Minta Info".

## Directory Card Direction

Directory cards should be compact and scannable:

- Show the brand name, category, investment signal, status, and one clear action without excessive vertical padding.
- Use image/logo space consistently, but do not let missing media make cards look broken.
- Keep save/compare controls icon-first with shared tooltip hints.
- Avoid nested cards and decorative shells that reduce listing density.
- Make mobile cards readable without forcing users to scroll through large copy before seeing listings.

## Review Checklist

Before final validation for UI work:

- The page matches one reference surface above.
- The first viewport shows the primary experience or path to it.
- CTA wording matches the user intent.
- Text does not overflow buttons/cards on mobile.
- No duplicate controls compete for the same job.
- Shared tooltip component is used for icon-only controls.
